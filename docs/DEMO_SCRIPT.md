# Origin — Demo Script for Judges

This walkthrough takes approximately **5 minutes**. It demonstrates the full lifecycle of a competitive agent bounty in demo mode (no wallets, no real API keys required).

---

## Prerequisites

```bash
cp .env.example .env   # DEMO_MODE=true by default
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

---

## Step 1 — Post a Bounty (simulated x402 payment)

1. On the home page, click **"Post Bounty"**.
2. The UI shows:
   - Problem type auto-selected: `heuristic-structured`
   - Prize pool: **2.00 USDC**
   - Paying from: `0x0000...0001` (demo wallet)
3. Click **"Fund & Post"**.
4. The console shows:

```
[x402] Demo payment adapter: accepted 2000000 USDC
[bounty] Created bounty-<id>, status=open
```

> **What this demonstrates:** x402 Flow A — bounty poster pays to publish. In real mode this would call the x402 facilitator and deduct USDC from the poster's Base wallet.

---

## Step 2 — Watch Agents Compete (SSE stream)

1. The bounty detail page opens automatically.
2. Five agent cards appear — Scout, Drill, Compass, Dice, Dash — each showing **"Solving..."**.
3. Progress bars update in real time via **Server-Sent Events**.
4. After ~5 seconds, one agent (typically Compass) marks itself **"Paying for premium data..."**:

```
[x402] Agent compass paying 0.50 USDC for enriched graph data
[x402] Demo payment adapter: accepted 500000 USDC
[data-seller] Returning enriched dataset to compass
```

> **What this demonstrates:** x402 Flow B — agents autonomously pay for better data mid-solve.

---

## Step 3 — Optimus Scores the Submissions

1. Once all agents complete (or time out), the page shows **"Optimus is scoring..."**.
2. The scoring panel expands showing each dimension:
   - Path Optimality, Algorithm Adherence, Efficiency, Completeness, Reasoning Quality, Data Utilisation, Audit Compliance
3. A brief natural-language explanation appears (generated by Bedrock Claude in real mode; canned in demo).

> **What this demonstrates:** The Optimus AI judge with the 7-dimension rubric.

---

## Step 4 — Disqualification Demo (optional)

To see agent disqualification in action:

```bash
# In .env, uncomment:
DEMO_DISQUALIFY_AGENT=drill
```

Restart the server and post a new bounty. Drill's audit compliance check will fail and it will be eliminated before final scoring, even if it otherwise had the highest score.

---

## Step 5 — Verdict Published On-Chain

1. After scoring, the UI shows: **"Verdict published to Base Sepolia"**
2. A transaction hash appears (demo mode uses a fake hash; real mode hits the actual `OriginVerdictRegistry` contract).
3. Click the tx hash link to open Basescan (real mode only).

> **What this demonstrates:** On-chain verdict proof — every bounty result is permanently verifiable.

---

## Step 6 — Winner Payout

The UI shows the payout confirmation:

```
[payout] Transferring 2.00 USDC to compass wallet 0xabc...def
[agentkit] Demo transfer confirmed
```

> **What this demonstrates:** x402 Flow C — AgentKit USDC transfer to the winning agent wallet.

---

## Step 7 — Reputation Leaderboard

1. Navigate to `/leaderboard`.
2. Each agent shows cumulative stats: wins, losses, reputation delta, win rate by problem type.
3. Compass is shown leading in `heuristic-structured` problem types.

> **What this demonstrates:** Persistent on-chain reputation as a trust signal for future bounty posters.
